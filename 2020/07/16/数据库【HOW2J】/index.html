¶11、检查MYSQL 是否开启INNODB，以及如何开启

¶步骤 1 : MYSQL 版本

本教材经实验可在5.1、5.5上使用，5.7默认是InnoDB的，所以无需额外开启

¶步骤 2 : 检查是否innodb已开启

执行命令

SHOW variables like ‘have_%’;

在结果中的have_innodb，如果显示为YES，即表示启动了。 如果是NO或者DISABLED表示未启动。
如果不存在也表示未启动。

¶步骤 3 : 停止mysql

首先执行命令停止mysql

net stop mysql

¶步骤 4 : 删除3个文件

定位到D:\tools\MYSQL\mysql-5.1.57-win32\data 目录下，找到这3个文件
ib_logfile0
ib_logfile1
ibdata1
备份后，删除掉

注： D:\tools\MYSQL\mysql-5.1.57-win32 是我的安装目录，请自行调整

¶步骤 5 : 在配置文件中增加一句

打开文件：D:\tools\MYSQL\mysql-5.1.57-win32\my.ini

添加一行

default-storage-engine=InnoDB

也许已经存在

default-storage-engine=MyISAM

像下面这样，注释掉即可

#default-storage-engine=MyISAM

注： D:\tools\MYSQL\mysql-5.1.57-win32 是我的安装目录，请自行调整

¶步骤 6 : 启动mysql

执行命令 net start

¶步骤 7 : 再检测

这个方法有些版本查不出来InnoDB是否开启 执行下面的语句 SELECT SUPPORT FROM INFORMATION_SCHEMA.ENGINES WHERE ENGINE = ‘InnoDB’; 如果innodb被启用并且是默认数据库，那么会出现

如果innodb可用，但不是默认引擎，则结果将是yes。如果没有，结果显然是no

¶13、MYSQL 安装失败，提示APPLY SECURITY SETTINGS 的处理办法

¶步骤 1 : mysql 卸载不干净

mysql 自带的卸载很多时候都卸载不干净，导致重新安装mysql的时候出现如图所示的问题。

接下来讲解如何手动完全卸载干净mysql

¶步骤 2 : mysql 卸载

运行[安装mysql-server] 右上角安装程序，点击Remove进行卸载。

如果当前Mysql在运行，会提示是否要关闭，点击YES

¶步骤 3 : 删除mysql安装目录

删除mysql目录

C:\Program Files\MySQL

¶步骤 4 : 删除my.ini

检查C:\WINDOWS目录下是否有my.ini文件,将其删除

这个文件不一定存在

¶步骤 5 : 运行注册表

win+r，弹出运行程序窗口，然后输入

regedit

¶步骤 6 : 删除key1

删除

HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\Eventlog\Application\MySQL

¶步骤 7 : 删除key2

如下两个key不一定有，没有就不理了
删除

HKEY_LOCAL_MACHINE\SYSTEM\ControlSet002\Services\Eventlog\Application \MySQL

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Eventlog\Application\MySQL

¶步骤 8 : 删除ProgramData目录下的文件

如果是win10, 则删除

C:\ProgramData\MySQL Server 5.5

这个目录默认是隐藏的，需要在文件夹选项中显示出来

如果是XP或者win7,对应的位置是：

C:\Documents and Settings\All Users\Application Data\MySQL

¶步骤 9 : 重新安装

运行[安装mysql-server]右上角安装程序，重新安装

¶第三章 hsqldb

¶1、简介

HSQLDB是一款Java内置的数据库，非常适合在用于快速的测试和演示的Java程序中。(无需独立安装数据库)

HSQLDB有三种模式：

  1. Server 就像Mysql那样
  2. In-Process 又叫做 Standalone 模式，数据放在本地文件，伴随JVM一起启动，是HSQLDB的主要应用场景
  3. Memory-only， 仅仅在内存中，一旦重启，数据就消失。

¶2、客户端

¶步骤 1 : 创建项目

首先创建一个java project，名字叫做hsqldb

¶步骤 2 : 导入jar包

在右上角下载lib.rar,解压出需要的jar包，并导入到项目中

¶步骤 3 : HSQLDB_Client

运行HSQLDB_Client，其主方法调用了DatabaseManagerSwing的主方法。。。。

这回导致启动一个Swing做的客户端

¶步骤 4 : 创建数据库，账号和密码

填写连接参数

  1. Setting Name：输入test
  2. Type: 选择HSQL Database Engine Standalone
  3. Driver: 使用默认的org.hsqldb.jdbcDriver
  4. URL： 输入 Jdbc:hsqldb:file:d:/hsqldb/test
  5. User: root
    6: Password:admin

第3步： 这是Standalone方式，对于同一个数据库，只能启动一次，不支持多进程连接同一个数据库，所以后续在写JAVA代码连接的时候，必须把这个客户端关闭掉才行。
第4步： Jdbc:hsqldb:file:d:/hsqldb/test会导致在d:/hsqldb目录下创建一个test数据库
第5,6步：会导致创建账号密码，以后再连接d:/hsqldb/下的test数据库，就必须用这个账号密码了。
